## What ReflectAOT does

ReflectAOT replaces **dynamic reflection** for selected operations with **build-time scans** and **generated bytecode**. Your application keeps calling `me.stringdotjar.reflectaot.Reflect`, but field access, property access, method dispatch, and method identity (`Reflect.method`) are resolved during the Gradle codegen task instead of via `java.lang.reflect` at runtime.

**Typical goals:** GraalVM Native Image, TeaVM/GWT-style transpilation, Android, or any environment where reflection is restricted or expensive.

---

## Gradle usage

Apply the published plugin (coordinates match your Gradle catalog or version catalog entry), enable the task on the variants that compile your sources, and ensure **`generateReflectAOT` (or the plugin’s registered task name) runs before** packaging or running the app that depends on generated classes.

Use the plugin extension to:

- Add **extra classes** whose bytecode was not scanned but must participate in dispatch (`extraClasses`).
- Tune **classpath roots** and **package excludes** if defaults hide legitimate types.

Exact task names and DSL blocks live next to `ReflectAOTPlugin.kt` and `ReflectAOTExtension.kt` in `reflectaot-gradle-plugin/` (see [Gradle plugin module](#gradle-plugin-module-reflectaot-gradle-plugin)).

## Runtime API: `Reflect`

All methods are on `me.stringdotjar.reflectaot.Reflect`. They delegate to `ReflectAOTServices`, which prefers the **generated registry** installed by `ReflectAOTBootstrap`, with safe fallbacks when codegen has not run.

### Comparison and copying

| Method | Purpose | Gotchas |
|--------|---------|---------|
| `compare(Object, Object)` | Stable ordering for nulls, numbers, strings; delegates other types to runtime. | Unsupported type pairs may throw `IllegalArgumentException`. |
| `compareMethods(int, int)` | True when two numeric ids denote the same scanned method. | Ids are meaningful only after codegen. |
| `compareMethods(ReflectMethodId, ReflectMethodId)` | Boxed id compare; null tokens throw. | — |
| `copy(T)` | Shallow copy when runtime supports it. | Often `UnsupportedOperationException` if not specialized. |

### Fields and properties

| Method | Purpose | Gotchas |
|--------|---------|---------|
| `field(o, name)` | Read a **declared field** by name. | Build must see a **concrete** receiver type (not only `Object`); name must match generated metadata. |
| `setField(o, name, value)` | Write a field. | Same receiver-type requirement. |
| `property(o, name)` | JavaBeans-style read (getter) with field fallback. | Property name is the **bean** name (e.g. `foo` for `getFoo()`), not the getter’s method name. |
| `setProperty(o, name, value)` | JavaBeans-style write with field fallback. | Same as above. |
| `hasField(o, name)` | True if field or property name is known. | — |
| `fields(o)` | Merged list of field and property names. | Never null; may be empty. |

### Method identity and invocation

| Method | Purpose | Gotchas |
|--------|---------|---------|
| `method(Class, String, String)` | Returns `ReflectMethodId` for a **fully specified** overload (JVM descriptor). | `Class`, name, and descriptor should be **compile-time constants** in user bytecode so the scanner can see them. |
| `method(Class, String)` | Same, but descriptor is **inferred** when exactly one public instance method with that name exists. | If multiple overloads exist, **the Gradle task fails**; add the 3-arg form with a descriptor. |
| `callMethod(o, id, List / varargs / ReflectMethodId)` | Invokes the bound method. | Arguments are taken from lists or varargs in **parameter order**; wrong arity throws at invocation. Void methods yield `null`. |

### Miscellaneous predicates

| Method | Purpose |
|--------|---------|
| `isEnumValue`, `isFunction`, `isObject` | Delegates to runtime defaults for parity with reflective ecosystems. |

## Generated artifacts

Under `me.stringdotjar.reflectaot.generated`:

| Type | Role |
|------|------|
| `ReflectAOTBootstrap` | Static initializer installs `ReflectAOTRegistry` and `ReflectAOTMethodIdTable` via `ReflectAOTServices`. |
| `ReflectAOTRegistry` | Implements `ReflectAOTRuntime`: `instanceof` dispatch to per-type accessors for field/property/`fields`/`callMethod`. |
| `access.<Type>_ReflectAOT` | Static helpers mirroring `Reflect.field`, `Reflect.property`, … for one concrete receiver type. |
| `ReflectAOTMethodIdTable` | Implements `ReflectAOTMethodIdResolver`: maps `(Class, name[, descriptor])` to `ReflectMethodId`. |

If you use **Java-only** output mode, `.java` mirrors are emitted instead of some `.class` files; behavior is intended to match.

## Codegen module (`reflectaot-codegen`)

Library used by the Gradle plugin: scans bytecode, validates against parsed class metadata, emits accessors/registry/method-id table/bootstrap.

### File reference

| File | Responsibility |
|------|----------------|
| `ReflectApiNames.kt` | **Central constants**: JVM method names on `Reflect` (`field`, `method`, …), sets for “needs concrete receiver” vs “full scan”, `Reflect.method` JVM descriptors, plus `ReflectClasspathScanDefaults` package-prefix excludes. |
| `ReflectAOTCodegen.kt` | Pipeline entry: merge excludes → scan → load types → validate names → bind method ids → emit bytecode and/or Java. |
| `ReflectUsageScanner.kt` | ASM tree walk: finds `Reflect` `invokestatic` sites, extracts receiver types from frames / heuristics, parses `Reflect.method` arguments from nearby instructions. |
| `ClasspathWalker.kt` | Enumerates `.class` files from directories and JARs. |
| `TypeIntrospection.kt` | Parses class files into fields, bean properties, public instance methods (superclass walk). |
| `JvmSubtype.kt` | Subtype check via superclass chain only (matches invokevirtual resolution for classes). |
| `MethodIdBinding.kt` | One resolved `Reflect.method` binding (id + declaring type + descriptor). |
| `ReflectAOTOutput.kt` | Enum: CLASS / JAVA / BOTH output mode. |
| `AccessBytecodeEmitter.kt` | Per-type accessor `.class` files. |
| `RegistryBytecodeEmitter.kt` | `ReflectAOTRegistry` bytecode. |
| `MethodIdTableBytecodeEmitter.kt` | `ReflectAOTMethodIdTable` bytecode (`resolve` overloads; uses `GeneratorAdapter.iinc` correctly for counters). |
| `MethodIdTableJavaEmitter.kt` | Java source mirror of the method-id table. |
| `BootstrapBytecodeEmitter.kt` | `ReflectAOTBootstrap` bytecode. |
| `JavaMirrorEmitter.kt` | Java mirrors for accessors + registry. |
| `JavaBootstrapEmitter.kt` | Java `ReflectAOTBootstrap`. |

Tests live under `reflectaot-codegen/src/test/kotlin` (e.g. `BootstrapGoldenTest`).

## Renaming Reflect APIs (`ReflectApiNames`)

If you rename a **public static** method on `Reflect.java`, update **`ReflectApiNames`** in `reflectaot-codegen` so that:

- The scanner still recognizes `invokestatic` targets.
- Generated accessors/registry/tables emit the correct method names.
- Error messages stay aligned (many interpolate `ReflectApiNames.METHOD` for `Reflect.method`).

Also search the **runtime** for hard-coded strings (e.g. `ReflectAOTServices` install messages) if users see them.

## Common errors and fixes

### `IllegalStateException: cannot infer concrete receiver types`

**Cause:** For `field`, `setField`, `property`, `setProperty`, `hasField`, `fields`, or `method`, the scanner could not prove a **concrete reference** type for the first argument (often everything is erased to `Object` in bytecode).

**Fix:** Cast or assign to a concrete type visible in bytecode, or add the type to `reflectaot` `extraClasses` so it is introspected.

### `IllegalStateException` about `Reflect.method` / overloads / unknown method

**Causes:**

- Two-arg `Reflect.method(Foo.class, "bar")` but several `bar` overloads exist.
- Class or method not public / not found in parsed bytecode.
- Non-constant arguments (scanner requires literals / `.class` it can see).

**Fix:** Use `Reflect.method(Foo.class, "bar", "(I)V")` with the exact JVM descriptor, or reduce overloads. Ensure the owning class is on the scan classpath.

### `UnsupportedOperationException: Reflect.field not specialized…`

**Cause:** Runtime dispatch has no generated accessor branch for the receiver’s **runtime** class (type not included in codegen inputs).

**Fix:** Rerun codegen with that type on the classpath; use `extraClasses` if it is only referenced indirectly.

### `IllegalArgumentException` / `VerifyError` around generated `ReflectAOTMethodIdTable`

Older codegen bugs produced invalid bytecode for `resolve(Class, String)` (bad `iinc`). Current codegen uses **`GeneratorAdapter.iinc`**. If you see verifier errors, ensure you are on a **fixed** version and **clean-rebuild** so stale `.class` files are replaced.

### `ClassNotFoundException: ReflectAOTBootstrap`

**Cause:** Generated output not on the runtime classpath, or task did not run.

**Fix:** Depend on the codegen output directory or the jar that embeds generated classes; run the generate task before run/package.

### Java mirror / registry mismatch

The Java emitter now names accessor methods **`property`** (not `getProperty`) to match `ReflectAOTRuntime` and bytecode. If you maintain a fork, keep those names aligned with `ReflectApiNames`.
